`<bs-clipboard>` wraps [clipboard.js](https://clipboardjs.com/) so you can drop it into any
HTML page like a regular custom element.

Wraps [clipboard.js](https://clipboardjs.com/) as a Lit-based web component. Re-dispatches the upstream lifecycle as standard `bs:*` custom events. Renders in **light DOM** so the upstream library's CSS reaches its children.

## Install

```sh
npm install @bootstrap-wc/plugin-clipboard clipboard
```

The plugin lists the upstream library as a peer dependency
(`clipboard@^2.0.11`) — install it alongside the wrapper.

## Loading

Load the upstream library and the plugin's `define` entry. Order matters —
the plugin reads `globalThis` for its dependencies on connect.

```html
<!-- Upstream library -->
<script src="https://cdn.jsdelivr.net/npm/clipboard@2"></script>

<!-- bootstrap-wc plugin (registers <bs-clipboard>) -->
<script type="module"
  src="https://esm.sh/@bootstrap-wc/plugin-clipboard/define"></script>
```

Or via npm:

```ts
import 'clipboard';
import '@bootstrap-wc/plugin-clipboard/define';
```

## Example

```html
<bs-clipboard>
  <button class="btn btn-outline-primary" data-clipboard-text="Copied!">Copy</button>
</bs-clipboard>
```

## Listening for events

```js
const el = document.querySelector('bs-clipboard');
el.addEventListener('bs:ready', (e) => {
  // e.detail.instance is the underlying clipboard.js instance
  console.log('ready', e.detail.instance);
});
```

## Manual control

Set `auto="false"` to suppress automatic instantiation, then drive the
lifecycle yourself.

```js
const el = document.querySelector('bs-clipboard');
el.setAttribute('auto', 'false');
el.options = { /* … */ };
el.instantiate();   // create the upstream instance
el.dispose();       // destroy it
```

## API

### Attributes / properties

| Name | Type | Default | Description |
|---|---|---|---|
| options | `object` | `{}` | Plain object passed as upstream library config (also accepts JSON string attribute). |
| auto | `boolean` | `true` | Auto-instantiate on connect. Set to `false` to opt into manual control. |

### Events

| Name | Detail | When |
|---|---|---|
| bs:ready | `{ instance }` | Fired after the upstream library is initialized. |
| bs:change | `{ instance, type, raw }` | Re-dispatched upstream events (e.g. `select`, `submit`). |
| bs:destroy | — | Fired before the upstream instance is torn down. |

### Slots

| Slot | Description |
|---|---|
| _(default)_ | Markup the upstream library operates on (canvas, table, input, etc.). |

## Compatibility

- Lives in **light DOM** so plugin CSS reaches the slotted children.
- Place it inside other `<bs-*>` components only when those expose **named
  slots** that participate in the light tree — shadow-root content is invisible
  to the upstream library.
- Avoid `document`-wide jQuery selectors that assume the plugin DOM is at the
  page root; scope to the wrapper instead.

See `packages/PLUGINS.md` in the source for the full plugin compatibility
matrix.
