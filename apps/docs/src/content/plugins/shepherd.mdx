`<bs-tour>` wraps [Shepherd.js](https://shepherdjs.dev/) so you can drop it into any
HTML page like a regular custom element.

Renders in **light DOM** so the upstream library's CSS reaches its slotted
children. Re-dispatches the upstream lifecycle as standard `bs:*` custom
events.

## Install

```sh
npm install @bootstrap-wc/plugin-shepherd shepherd.js
```

The plugin lists the upstream library as a peer dependency
(`shepherd.js@^13.0.3`) — install it alongside the wrapper.

## Loading

Load the upstream library and the plugin's `define` entry. Order matters —
the plugin reads `globalThis` for its dependencies on connect.

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/shepherd.js@13.0.3/dist/css/shepherd.css">
<script src="https://cdn.jsdelivr.net/npm/shepherd.js@13.0.3/dist/js/shepherd.min.js"></script>

<!-- bootstrap-wc plugin (registers <bs-tour>) -->
<script type="module"
  src="https://esm.sh/@bootstrap-wc/plugin-shepherd/define"></script>
```

Or via npm:

```ts
import 'shepherd.js';
import '@bootstrap-wc/plugin-shepherd/define';
```

## Listening for events

```js
const el = document.querySelector('bs-tour');
el.addEventListener('bs:ready', (e) => {
  // e.detail.instance is the underlying Shepherd.js instance
  console.log('ready', e.detail.instance);
});
```

## Manual control

Set `auto="false"` to suppress automatic instantiation, then drive the
lifecycle yourself.

```js
const el = document.querySelector('bs-tour');
el.setAttribute('auto', 'false');
el.options = { /* … */ };
el.connectedCallback();   // re-runs instantiate when auto is later flipped
```

## API

### Attributes / properties

| Name | Type | Default | Description |
|---|---|---|---|
| options | `object` | `{}` | Plain object passed as upstream library config (also accepts JSON string attribute). |
| auto | `boolean` | `true` | Auto-instantiate on connect. Set to `false` to opt into manual control. |

### Events

| Name | Detail | When |
|---|---|---|
| bs:ready | `{ instance, target }` | Fired after the upstream library is initialized. |
| bs:destroy | — | Fired before the upstream instance is torn down. |

### Slots

| Slot | Description |
|---|---|
| _(default)_ | Markup the upstream library operates on (canvas, table, input, etc.). |

## Compatibility

- Lives in **light DOM** so plugin CSS reaches the slotted children.
- Place it inside other `<bs-*>` components only when those expose **named
  slots** that participate in the light tree — shadow-root content is invisible
  to the upstream library.
- Avoid `document`-wide jQuery selectors that assume the plugin DOM is at the
  page root; scope to the wrapper instead.
