`<bs-i18n>` wraps [i18next](https://www.i18next.com/) so you can drop it into any
HTML page like a regular custom element.

Renders the translation for `key` after `i18next.init()` has resolved. Updates when the language changes.

Wraps [i18next](https://www.i18next.com/) and re-dispatches lifecycle as standard `bs:*` custom events. Renders in **light DOM** so the upstream library's CSS reaches its children.

## Live example

<Example code={`<p>
  <bs-i18n key="welcome.title"></bs-i18n><br>
  <bs-i18n key="welcome.subtitle"></bs-i18n>
</p>

        fr: { translation: { welcome: { title: 'Bienvenue !', subtitle: 'Vous lisez la documentation.' } } },
      },
    });
  }
</script>`} />

## Install

```sh
npm install @bootstrap-wc/plugin-i18next i18next
```

The plugin lists the upstream library as a peer dependency
(`i18next@^23.15.1`) — install it alongside the wrapper.

## Loading

Load the upstream library and the plugin's `define` entry. Order matters —
the plugin reads `globalThis` for its dependencies on connect.

```html
<script src="https://cdn.jsdelivr.net/npm/i18next@23.15.1/i18next.min.js"></script>

<!-- bootstrap-wc plugin (registers <bs-i18n>) -->
<script type="module"
  src="https://esm.sh/@bootstrap-wc/plugin-i18next/define"></script>
```

Or via npm:

```ts
import 'i18next';
import '@bootstrap-wc/plugin-i18next/define';
```

## Listening for events

```js
const el = document.querySelector('bs-i18n');
el.addEventListener('bs:ready', (e) => {
  // e.detail.instance is the underlying i18next instance
  console.log('ready', e.detail.instance);
});
```

## Manual control

Set `auto="false"` to suppress automatic instantiation, then drive the
lifecycle yourself.

```js
const el = document.querySelector('bs-i18n');
el.setAttribute('auto', 'false');
el.options = { /* … */ };
el.instantiate();   // create the upstream instance
el.dispose();       // destroy it
```

## API

### Attributes / properties

| Name | Type | Default | Description |
|---|---|---|---|
| key | `string` | — | i18next translation key. |
| options | `object` | `{}` | Per-instance interpolation options passed to t(). |
| auto | `boolean` | `true` | Auto-translate on connect / language change. |

### Events

| Name | Detail | When |
|---|---|---|
| bs:ready | `{ instance }` | Fired after the upstream library is initialized. |
| bs:change | `{ instance, type, raw }` | Re-dispatched upstream events (e.g. `select`, `submit`). |
| bs:destroy | — | Fired before the upstream instance is torn down. |

### Slots

| Slot | Description |
|---|---|
| _(default)_ | Markup the upstream library operates on (canvas, table, input, etc.). |

## Compatibility

- Lives in **light DOM** so plugin CSS reaches the slotted children.
- Place it inside other `<bs-*>` components only when those expose **named
  slots** that participate in the light tree — shadow-root content is invisible
  to the upstream library.
- Avoid `document`-wide jQuery selectors that assume the plugin DOM is at the
  page root; scope to the wrapper instead.

See `packages/PLUGINS.md` in the source for the full plugin compatibility
matrix.
