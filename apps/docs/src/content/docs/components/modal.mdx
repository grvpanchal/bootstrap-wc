A modal dialog with backdrop, focus trap, and keyboard dismissal — implemented with [`focus-trap`](https://github.com/focus-trap/focus-trap).

## Modal components

Below is a **static** modal rendering (meaning its `position` is overridden and it's shown inline). Use the `static-preview` attribute so the dialog sits in normal document flow with no backdrop — handy for visual previews without triggering the show/hide lifecycle.

<Example code={`<bs-modal static-preview heading="Modal title">
  <p>Modal body text goes here.</p>
  <div slot="footer">
    <bs-button variant="secondary">Close</bs-button>
    <bs-button variant="primary">Save changes</bs-button>
  </div>
</bs-modal>`} />

## Modal title

A real modal is launched from a trigger — clicking the button below calls `show()` on the `bs-modal` via an inline handler. This live demo matches Bootstrap's **Live demo**, **Static backdrop**, and **Scrolling long content** examples.

<Example code={`<bs-modal id="demo-live" heading="Modal title">
  <p>Woo-hoo, you're reading this text in a modal!</p>
  <div slot="footer">
    <bs-button variant="secondary" onclick="document.getElementById('demo-live').hide()">Close</bs-button>
    <bs-button variant="primary">Save changes</bs-button>
  </div>
</bs-modal>
<bs-button variant="primary" onclick="document.getElementById('demo-live').show()">Launch demo modal</bs-button>`} />

When the backdrop is static, clicking outside the modal won't close it. Pair `static-backdrop` with `no-close-on-escape` if you also want to disable ESC-to-close.

<Example code={`<bs-modal id="demo-static" static-backdrop no-close-on-escape heading="Modal title">
  <p>I will not close if you click outside of me. Don't even try to press escape key.</p>
  <div slot="footer">
    <bs-button variant="secondary" onclick="document.getElementById('demo-static').hide()">Close</bs-button>
    <bs-button variant="primary">Understood</bs-button>
  </div>
</bs-modal>
<bs-button variant="primary" onclick="document.getElementById('demo-static').show()">Launch static backdrop modal</bs-button>`} />

## Tooltips and popovers

Tooltips and popovers can be placed within modals as needed. When a modal is closed, any tooltips and popovers within are also automatically dismissed.

<Example code={`<bs-modal id="demo-popovers" heading="Modal title">
  <h2 class="fs-5">Popover in a modal</h2>
  <p>This <bs-button variant="secondary">button</bs-button> triggers a popover on click.</p>
  <hr/>
  <h2 class="fs-5">Tooltips in a modal</h2>
  <p><a href="#">This link</a> and <a href="#">that link</a> have tooltips on hover.</p>
  <div slot="footer">
    <bs-button variant="secondary" onclick="document.getElementById('demo-popovers').hide()">Close</bs-button>
    <bs-button variant="primary">Save changes</bs-button>
  </div>
</bs-modal>
<bs-button variant="primary" onclick="document.getElementById('demo-popovers').show()">Launch demo modal</bs-button>`} />

## Grids in modals

Utilize the Bootstrap grid system within a modal by nesting `.container-fluid` within the modal body. Then, use the normal grid system classes as you would anywhere else.

<Example code={`<bs-modal id="demo-grid" heading="Grids in modals">
  <div class="container-fluid bd-example-row">
    <div class="row">
      <div class="col-md-4">.col-md-4</div>
      <div class="col-md-4 ms-auto">.col-md-4 .ms-auto</div>
    </div>
    <div class="row">
      <div class="col-md-3 ms-auto">.col-md-3 .ms-auto</div>
      <div class="col-md-2 ms-auto">.col-md-2 .ms-auto</div>
    </div>
    <div class="row">
      <div class="col-md-6 ms-auto">.col-md-6 .ms-auto</div>
    </div>
    <div class="row">
      <div class="col-sm-9">
        Level 1: .col-sm-9
        <div class="row">
          <div class="col-8 col-sm-6">Level 2: .col-8 .col-sm-6</div>
          <div class="col-4 col-sm-6">Level 2: .col-4 .col-sm-6</div>
        </div>
      </div>
    </div>
  </div>
</bs-modal>
<bs-button variant="primary" onclick="document.getElementById('demo-grid').show()">Launch demo modal</bs-button>`} />

## Varying modal content

<Example code={`<bs-modal id="demo-varying" heading="New message">

      <label for="recipient-name" class="col-form-label">Recipient:</label>

    <div class="mb-3">
      <label for="message-text" class="col-form-label">Message:</label>
      <bs-input id="message-text" type="text"></bs-input>

## Toggle between modals

Toggle between multiple modals by opening the next one and closing the current. Swap content by chaining `show()` on the target modal and `hide()` on the current.

<Example code={`<bs-modal id="demo-toggle-1" centered heading="Modal 1">
  Show a second modal and hide this one with the button below.
  <div slot="footer">
    <bs-button variant="primary" onclick="document.getElementById('demo-toggle-1').hide(); document.getElementById('demo-toggle-2').show()">Open second modal</bs-button>
  </div>
</bs-modal>
<bs-modal id="demo-toggle-2" centered heading="Modal 2">
  Hide this modal and show the first with the button below.
  <div slot="footer">
    <bs-button variant="primary" onclick="document.getElementById('demo-toggle-2').hide(); document.getElementById('demo-toggle-1').show()">Back to first</bs-button>
  </div>
</bs-modal>
<bs-button variant="primary" onclick="document.getElementById('demo-toggle-1').show()">Open first modal</bs-button>`} />

Modals have three optional sizes, available via the `size` attribute. These sizes kick in at certain breakpoints to avoid horizontal scrollbars on narrower viewports.

<bs-modal id="demo-lg" size="lg" heading="Large modal">Large modal body.</bs-modal>

## Fullscreen Modal

Another override is the option to pop up a modal that covers the user viewport, available via the `fullscreen` attribute. It accepts a boolean (always fullscreen) or a breakpoint modifier (`sm-down`, `md-down`, `lg-down`, `xl-down`, `xxl-down`) to only go fullscreen below the chosen breakpoint.

## API

### Attributes / properties

| Name | Type | Default | Description |
|---|---|---|---|
| `open` | `boolean` | `false` | Controls visibility. Reflected. |
| `heading` | `string` | — | Title text (slot `title` available for rich content). |
| `size` | `sm` \| `md` \| `lg` \| `xl` | `md` | Dialog width. |
| `centered` | `boolean` | `false` | Vertically center the dialog. |
| `scrollable` | `boolean` | `false` | Scrollable body. |
| `static-backdrop` | `boolean` | `false` | Clicking the backdrop doesn't close the modal. |
| `no-backdrop` | `boolean` | `false` | Hide the backdrop entirely. |
| `no-close-on-escape` | `boolean` | `false` | Disable ESC-to-close. |
| `no-close-button` | `boolean` | `false` | Hide the header close button. |
| `fullscreen` | `boolean` \| `sm-down` \| `md-down` \| `lg-down` \| `xl-down` \| `xxl-down` | `false` | Apply `.modal-fullscreen` (or breakpoint-scoped variant). |
| `static-preview` | `boolean` | `false` | Render the dialog inline (for docs previews); suppresses backdrop and fixed positioning. |

### Methods

| Method | Description |
|---|---|
| `show()` | Open the modal. |
| `hide()` | Close the modal. |
| `toggle()` | Toggle open state. |

### Events

| Event | Description |
|---|---|
| `bs-show` | Before the modal opens. Cancelable. |
| `bs-shown` | After open animation completes. |
| `bs-hide` | Before the modal closes. Cancelable. |
| `bs-hidden` | After close animation completes. |

### Slots

| Slot | Description |
|---|---|
| default | Modal body content. |
| `title` | Rich title markup (overrides `heading`). |
| `footer` | Modal footer. Header is omitted if empty. |
