Block- and inline-level help text for form controls, plus server-rendered
valid / invalid feedback. Mirrors Bootstrap's
[Form text](https://getbootstrap.com/docs/5.3/forms/form-control/#form-text)
utility.

## Example

<Example code={`<bs-form-text>We never share your email with anyone else.</bs-form-text>`} />

## Form text

Block-level help text in forms can be created using `<bs-form-text>`.
Associate the help text with its input via `aria-describedby`, pointing at
the `id` on the `<bs-form-text>`. This ensures assistive technologies
(screen readers) announce the help text when users focus or enter the
control.

<Example code={`<bs-form-label for="inputPassword5">Password</bs-form-label>
<bs-input
  type="password"
  id="inputPassword5"
  aria-describedby="passwordHelpBlock"
></bs-input>
<bs-form-text id="passwordHelpBlock">
  Your password must be 8-20 characters long, contain letters and numbers,
  and must not contain spaces, special characters, or emoji.
</bs-form-text>`} />

Inline text uses the `inline` attribute to keep the help text on the same
line as its control — useful for compact grid layouts where the label,
input, and help text sit in `col-auto` columns.

<Example code={`<div class="row g-3 align-items-center">
  <div class="col-auto">
    <bs-form-label for="inputPassword6" class="col-form-label">
      Password
    </bs-form-label>
  </div>
  <div class="col-auto">
    <bs-input
      type="password"
      id="inputPassword6"
      aria-describedby="passwordHelpInline"
    ></bs-input>
  </div>
  <div class="col-auto">
    <bs-form-text inline id="passwordHelpInline">
      Must be 8-20 characters long.
    </bs-form-text>
  </div>
</div>`} />

## Valid and invalid feedback

Set `kind="valid"` or `kind="invalid"` to render Bootstrap's validation
feedback styles instead of the neutral help-text style. Feedback is only
shown by Bootstrap's CSS when the associated control has the matching
`.is-valid` / `.is-invalid` class (or sits inside a `.was-validated` form).

<Example code={`<bs-form-label for="validInput">Email</bs-form-label>
<bs-input id="validInput" class="is-valid" value="ok@example.com"></bs-input>
<bs-form-text kind="valid">Looks good!</bs-form-text>`} />

<Example code={`<bs-form-label for="invalidInput">Email</bs-form-label>
<bs-input id="invalidInput" class="is-invalid" value="nope"></bs-input>
<bs-form-text kind="invalid">Please provide a valid email.</bs-form-text>`} />

## Install

```sh
# as an npm package
npm install @bootstrap-wc/components
# or copy the source
npx bwc add form-text
```

## Usage

```ts
import '@bootstrap-wc/components/form-text';
```

```html
<bs-form-text>We never share your email.</bs-form-text>
```

## API

| Attribute | Type                               | Default  | Description |
| --------- | ---------------------------------- | -------- | ----------- |
| `kind`    | `'help' \| 'valid' \| 'invalid'`   | `'help'` | Which Bootstrap class to apply to the host: `.form-text`, `.valid-feedback`, or `.invalid-feedback`. |
| `inline`  | `boolean`                          | `false`  | Render the host inline (`display: inline`) instead of block-level. Use inside flex rows where help text shares the line with its control. |

The host element carries the relevant Bootstrap class (`form-text`,
`valid-feedback`, `invalid-feedback`); give it an `id` and reference it
from the associated input's `aria-describedby` attribute.

See `packages/components/src/form-text/` in the source for the complete
property / event list.
