`<bs-range>` is a **form-associated custom element** that mirrors Bootstrap's
`.form-range` styling. It participates in native `<form>` submission, reset,
and validation just like a plain `<input type="range">`.

## Overview

Create custom `<input type="range">` controls by pairing a `<bs-form-label>`
with a `<bs-range>`.

<Example code={`<bs-form-label for="customRange1">Example range</bs-form-label>
<bs-range id="customRange1"></bs-range>`} />

## Disabled

Add the `disabled` boolean attribute on an input to give it a greyed out
appearance and remove pointer events.

<Example code={`<bs-form-label for="disabledRange">Disabled range</bs-form-label>
<bs-range id="disabledRange" disabled></bs-range>`} />

## Min and max

Range inputs have implicit values for `min` and `max` — `0` and `100`,
respectively. You may specify new values for those using the `min` and `max`
attributes.

<Example code={`<bs-form-label for="customRange2">Example range</bs-form-label>
<bs-range id="customRange2" min="0" max="5"></bs-range>`} />

## Steps

By default, range inputs "snap" to integer values. To change this, you can
specify a `step` value. In the example below, we double the number of steps
by using `step="0.5"`.

<Example code={`<bs-form-label for="customRange3">Example range</bs-form-label>
<bs-range id="customRange3" min="0" max="5" step="0.5"></bs-range>`} />

## Output value

Listen for the `bs-input` event and mirror `event.detail.value` into an
`<output>` element to display the current value of the range as the user
drags the thumb.

<Example code={`<bs-form-label for="customRange4">Example range</bs-form-label>
<bs-range id="customRange4" min="0" max="100" value="50"></bs-range>
<output for="customRange4" id="rangeValue" aria-hidden="true">50</output>`} />

```js
const rangeInput = document.getElementById('customRange4');
const rangeOutput = document.getElementById('rangeValue');
rangeOutput.textContent = rangeInput.value;
rangeInput.addEventListener('bs-input', (e) => {
  rangeOutput.textContent = e.detail.value;
});
```

## Install

```sh
# as an npm package
npm install @bootstrap-wc/components
# or copy the source
npx bwc add range
```

## Usage

```ts
import '@bootstrap-wc/components/range';
```

```html
<bs-range value="40"></bs-range>
```

## API

### Attributes / properties

| Name | Type | Default | Description |
|---|---|---|---|
| `value` | `string` | `'50'` | Current value. |
| `min` | `number` | `0` | Minimum value. |
| `max` | `number` | `100` | Maximum value. |
| `step` | `number` | `1` | Step increment. |
| `name` | `string` | `''` | Form field name. |
| `disabled` | `boolean` | `false` | Disables the slider. |

### Events

| Event | Detail | Description |
|---|---|---|
| `bs-input` | `{ value }` | Fires while the thumb is dragged. |

### Forms

```html
<form onsubmit="event.preventDefault(); console.log(new FormData(event.target))">
  <bs-range name="volume" value="25"></bs-range>
  <bs-button type="submit">Submit</bs-button>
</form>
```

The submitted FormData will contain `volume` — no JS state management needed.

> Safari < 16.4 lacks `ElementInternals`. Test your form flows on older Safari if you support it.
