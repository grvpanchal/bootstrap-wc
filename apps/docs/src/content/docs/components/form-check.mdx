The `<bs-form-check>` element renders a Bootstrap checkbox, radio, or switch control. Use `type="switch"` for the toggle style; use `type="radio"` with a shared `name` to group radios. `<bs-checkbox>`, `<bs-radio>`, and `<bs-switch>` are ergonomic aliases that default `type` for you.

## Checks

<Example code={`<bs-form-check label="Default checkbox"></bs-form-check>
<bs-form-check label="Checked checkbox" checked></bs-form-check>`} />

### Indeterminate

Set the `indeterminate` attribute to put a checkbox in the indeterminate state. User interaction clears it, mirroring native behavior.

<Example code={`<bs-form-check label="Indeterminate checkbox" indeterminate></bs-form-check>`} />

### Disabled

<Example code={`<bs-form-check label="Disabled indeterminate checkbox" indeterminate disabled></bs-form-check>
<bs-form-check label="Disabled checkbox" disabled></bs-form-check>
<bs-form-check label="Disabled checked checkbox" checked disabled></bs-form-check>`} />

## Radios

Radios with the same `name` form a mutually exclusive group.

<Example code={`<bs-form-check type="radio" name="radioDefault" label="Default radio"></bs-form-check>
<bs-form-check type="radio" name="radioDefault" label="Default checked radio" checked></bs-form-check>`} />

### Disabled

<Example code={`<bs-form-check type="radio" name="radioDisabled" label="Disabled radio" disabled></bs-form-check>
<bs-form-check type="radio" name="radioDisabled" label="Disabled checked radio" checked disabled></bs-form-check>`} />

## Switches

Switches use the switch role with checkbox semantics. Use `type="switch"` (or `<bs-switch>`).

<Example code={`<bs-form-check type="switch" label="Default switch checkbox input"></bs-form-check>
<bs-form-check type="switch" label="Checked switch checkbox input" checked></bs-form-check>
<bs-form-check type="switch" label="Disabled switch checkbox input" disabled></bs-form-check>
<bs-form-check type="switch" label="Disabled checked switch checkbox input" checked disabled></bs-form-check>`} />

## Default (stacked)

Checks and radios stack vertically by default.

<Example code={`<bs-form-check label="Default checkbox"></bs-form-check>
<bs-form-check label="Disabled checkbox" disabled></bs-form-check>`} />

<Example code={`<bs-form-check type="radio" name="exampleRadios" value="option1" label="Default radio" checked></bs-form-check>
<bs-form-check type="radio" name="exampleRadios" value="option2" label="Second default radio"></bs-form-check>
<bs-form-check type="radio" name="exampleRadios" value="option3" label="Disabled radio" disabled></bs-form-check>`} />

## Inline

Add the `inline` attribute to lay checks or radios out horizontally.

<Example code={`<bs-form-check inline value="option1" label="1"></bs-form-check>
<bs-form-check inline value="option2" label="2"></bs-form-check>
<bs-form-check inline value="option3" label="3 (disabled)" disabled></bs-form-check>`} />

<Example code={`<bs-form-check type="radio" inline name="inlineRadioOptions" value="option1" label="1"></bs-form-check>
<bs-form-check type="radio" inline name="inlineRadioOptions" value="option2" label="2"></bs-form-check>
<bs-form-check type="radio" inline name="inlineRadioOptions" value="option3" label="3 (disabled)" disabled></bs-form-check>`} />

## Reverse

Put the control after the label with the `reverse` attribute.

<Example code={`<bs-form-check reverse label="Reverse checkbox"></bs-form-check>
<bs-form-check reverse label="Disabled reverse checkbox" disabled></bs-form-check>

<bs-form-check type="switch" reverse label="Reverse switch checkbox input"></bs-form-check>`} />

## Without labels

Omit `label` and pass `aria-label` for visual-only controls.

<Example code={`<bs-form-check aria-label="..."></bs-form-check>
<bs-form-check type="radio" name="radioNoLabel" aria-label="..."></bs-form-check>`} />

## Toggle buttons

For button-styled toggles (Bootstrap's `btn-check` pattern), see [`<bs-button>`](../button/) with the appropriate pressed state. Toggle buttons are not part of `<bs-form-check>`.

## API

### Attributes / properties

| Name | Type | Default | Description |
|---|---|---|---|
| `type` | `checkbox` \| `radio` \| `switch` | `checkbox` | Control variant. |
| `checked` | `boolean` | `false` | Whether the control is checked. Reflected. |
| `indeterminate` | `boolean` | `false` | Visually indeterminate state (checkboxes). Cleared on user interaction. |
| `disabled` | `boolean` | `false` | Disables interaction. Reflected. |
| `required` | `boolean` | `false` | Marks the control as required for form submission. Reflected. |
| `name` | `string` | — | Form field name. Radios with the same `name` form a group. |
| `value` | `string` | `on` | Form value when checked. |
| `label` | `string` | — | Text label. Omit to render label-less (pair with `aria-label`). |
| `aria-label` | `string` | — | Accessible name when no visible label is rendered. |
| `inline` | `boolean` | `false` | Lay out inline with siblings. |
| `reverse` | `boolean` | `false` | Place the control after the label. |
| `invalid` | `boolean` | `false` | Applies `.is-invalid` styling. |
| `valid` | `boolean` | `false` | Applies `.is-valid` styling. |

### Events

| Event | Detail | Description |
|---|---|---|
| `bs-change` | `{ checked, value }` | Fired on user toggle. |

### Shadow parts

| Part | Description |
|---|---|
| `wrapper` | Outer `.form-check` wrapper. |
| `input` | Inner `<input>` element. |
| `label` | Inner `<label>` element (when rendered). |

## Install

```sh
# as an npm package
npm install @bootstrap-wc/components
# or copy the source
npx bwc add form-check
```

## Usage

```ts
import '@bootstrap-wc/components/form-check';
```

```html
<bs-form-check label="Check me"></bs-form-check>
<bs-form-check type="switch" label="Switch on"></bs-form-check>
<bs-form-check type="radio" name="group" label="Pick me"></bs-form-check>
```
