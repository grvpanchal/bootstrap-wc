title: Input group

`<bs-input-group>` wraps a form control with addons (text, buttons, icons). Mirror Bootstrap's layout by slotting `<bs-input-text>` addons, `<button class="btn">` addons, and real `<input class="form-control">` / `<select class="form-select">` form controls as direct children — the host carries `.input-group` so Bootstrap's sibling selectors resolve across the shadow boundary.

## Basic example

Place one addon on either (or both) sides of an input. On the right side, the input gets `aria-describedby` to announce extra content to assistive technologies.

<Example code={`<bs-input-group class="mb-3">
  <bs-input-text id="basic-addon1">@</bs-input-text>
  <input type="text" class="form-control" placeholder="Username" aria-label="Username" aria-describedby="basic-addon1">
</bs-input-group>

<bs-input-group class="mb-3">
  <input type="text" class="form-control" placeholder="Recipient's username" aria-label="Recipient's username" aria-describedby="basic-addon2">
  <bs-input-text id="basic-addon2">@example.com</bs-input-text>
</bs-input-group>

<div class="mb-3">
  <label for="basic-url" class="form-label">Your vanity URL</label>
  <bs-input-group>
    <bs-input-text id="basic-addon3">https://example.com/users/</bs-input-text>
    <input type="text" class="form-control" id="basic-url" aria-describedby="basic-addon3 basic-addon4">
  </bs-input-group>
  <div class="form-text" id="basic-addon4">Example help text goes outside the input group.</div>
</div>

<bs-input-group class="mb-3">
  <bs-input-text>$</bs-input-text>
  <input type="text" class="form-control" aria-label="Amount (to the nearest dollar)">
  <bs-input-text>.00</bs-input-text>
</bs-input-group>

<bs-input-group class="mb-3">
  <input type="text" class="form-control" placeholder="Username" aria-label="Username">
  <bs-input-text>@</bs-input-text>
  <input type="text" class="form-control" placeholder="Server" aria-label="Server">
</bs-input-group>

<bs-input-group>
  <bs-input-text>With textarea</bs-input-text>
  <textarea class="form-control" aria-label="With textarea"></textarea>
</bs-input-group>`} />

## Wrapping

Input groups wrap by default via `flex-wrap: wrap`, in order to accommodate custom form field validation within an input group. Disable this with `.flex-nowrap`.

<Example code={`<bs-input-group class="flex-nowrap">
  <bs-input-text id="addon-wrapping">@</bs-input-text>
  <input type="text" class="form-control" placeholder="Username" aria-label="Username" aria-describedby="addon-wrapping">
</bs-input-group>`} />

## Sizing

Add the `size` attribute to scale every control inside the group. Sizing on the individual input or text elements isn't supported — the group sets the scale.

<Example code={`<bs-input-group size="sm" class="mb-3">
  <bs-input-text id="inputGroup-sizing-sm">Small</bs-input-text>
  <input type="text" class="form-control" aria-label="Sizing example input" aria-describedby="inputGroup-sizing-sm">
</bs-input-group>

<bs-input-group class="mb-3">
  <bs-input-text id="inputGroup-sizing-default">Default</bs-input-text>
  <input type="text" class="form-control" aria-label="Sizing example input" aria-describedby="inputGroup-sizing-default">
</bs-input-group>

<bs-input-group size="lg">
  <bs-input-text id="inputGroup-sizing-lg">Large</bs-input-text>
  <input type="text" class="form-control" aria-label="Sizing example input" aria-describedby="inputGroup-sizing-lg">
</bs-input-group>`} />

## Checkboxes and radios

Use checkboxes and radios in place of text in input groups by nesting a `.form-check-input` inside a `<bs-input-text>`.

<Example code={`<bs-input-group class="mb-3">
  <bs-input-text>
    <input class="form-check-input mt-0" type="checkbox" value="" aria-label="Checkbox for following text input">
  </bs-input-text>
  <input type="text" class="form-control" aria-label="Text input with checkbox">
</bs-input-group>

<bs-input-group>
  <bs-input-text>
    <input class="form-check-input mt-0" type="radio" value="" aria-label="Radio button for following text input">
  </bs-input-text>
  <input type="text" class="form-control" aria-label="Text input with radio button">
</bs-input-group>`} />

## Multiple inputs

While multiple inputs are supported visually, validation styles are only available for input groups with a single input.

<Example code={`<bs-input-group>
  <bs-input-text>First and last name</bs-input-text>
  <input type="text" aria-label="First name" class="form-control">
  <input type="text" aria-label="Last name" class="form-control">
</bs-input-group>`} />

## Multiple addons

Multiple add-ons are supported and can be mixed with checkbox and radio input versions.

<Example code={`<bs-input-group class="mb-3">
  <bs-input-text>$</bs-input-text>
  <bs-input-text>0.00</bs-input-text>
  <input type="text" class="form-control" aria-label="Dollar amount (with dot and two decimal places)">
</bs-input-group>

<bs-input-group>
  <input type="text" class="form-control" aria-label="Dollar amount (with dot and two decimal places)">
  <bs-input-text>$</bs-input-text>
  <bs-input-text>0.00</bs-input-text>
</bs-input-group>`} />

## Install

```sh
# as an npm package
npm install @bootstrap-wc/components
# or copy the source
npx bwc add input-group
```

## Usage

```ts
import '@bootstrap-wc/components/input-group';
```

```html
<bs-input-group>
  <bs-input-text>$</bs-input-text>
  <input type="text" class="form-control" placeholder="Amount">
</bs-input-group>
```

## API

### `<bs-input-group>` attributes

| Name | Type | Default | Description |
|---|---|---|---|
| `size` | `sm` \| `md` \| `lg` | `md` | Scales every control inside via `.input-group-sm` / `.input-group-lg`. |
| `has-validation` | `boolean` | `false` | Applies `.has-validation` so validation feedback layouts correctly with a trailing addon. |

### `<bs-input-text>` attributes

`<bs-input-text>` takes no attributes beyond the host's global ones. The host carries `.input-group-text`; slot its addon label as the default child.

See `packages/components/src/input-group/` for the source.
