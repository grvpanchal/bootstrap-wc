Flexible content container for displaying a variety of content — text, images,
list groups, headers, and footers.

## Example

<Example code={`<div style="width: 18rem;">
  <bs-card heading="Card title">
    <svg slot="image" class="card-img-top" width="100%" height="180" xmlns="http://www.w3.org/2000/svg" role="img" aria-label="Placeholder: Image cap" preserveAspectRatio="xMidYMid slice"><title>Placeholder</title><rect width="100%" height="100%" fill="#868e96"></rect><text x="50%" y="50%" fill="#dee2e6" dy=".3em">Image cap</text></svg>
    <p class="card-text">Some quick example text to build on the card title and make up the bulk of the card's content.</p>
    <a href="#" class="btn btn-primary">Go somewhere</a>
  </bs-card>
</div>`} />

## Install

```sh
# as an npm package
npm install @bootstrap-wc/components
# or copy the source
npx bwc add card
```

## Usage

```ts
import '@bootstrap-wc/components/card';
```

```html
<bs-card heading="Card title" subtitle="Subtitle">Some quick example text.</bs-card>
```

## Body

The building block of a card is the `card-body`. By default `<bs-card>`
renders its default slot inside a `.card-body` so plain text "just works".

<Example code={`<bs-card>This is some text within a card body.</bs-card>`} />

## Titles, text, and links

Pass `heading` and `subtitle` attributes for titled cards, or write your own
`<h5 class="card-title">` and use `card-link` anchors inside.

<Example code={`<div style="width: 18rem;">
  <bs-card heading="Card title" subtitle="Card subtitle">
    <p class="card-text">Some quick example text to build on the card title and make up the bulk of the card's content.</p>
    <a href="#" class="card-link">Card link</a>
    <a href="#" class="card-link">Another link</a>
  </bs-card>
</div>`} />

## Images

The `image` slot renders a `.card-img-top`; the `image-bottom` slot renders
a `.card-img-bottom`.

<Example code={`<div style="width: 18rem;">

## List groups

Use `no-body` to opt out of the default `card-body` wrapper and put a
`.list-group.list-group-flush` directly inside the card.

<Example code={`<div style="width: 18rem;">
  <bs-card no-body>
    <ul class="list-group list-group-flush">
      <li class="list-group-item">An item</li>
      <li class="list-group-item">A second item</li>
      <li class="list-group-item">A third item</li>
    </ul>
  </bs-card>
</div>`} />

Combine a header slot with a flush list group.

<Example code={`<div style="width: 18rem;">
  <bs-card no-body>
    <span slot="header">Featured</span>
    <ul class="list-group list-group-flush">
      <li class="list-group-item">An item</li>
      <li class="list-group-item">A second item</li>
      <li class="list-group-item">A third item</li>
    </ul>
  </bs-card>
</div>`} />

Or combine a flush list group with a footer.

<Example code={`<div style="width: 18rem;">
  <bs-card no-body>
    <ul class="list-group list-group-flush">
      <li class="list-group-item">An item</li>
      <li class="list-group-item">A second item</li>
      <li class="list-group-item">A third item</li>
    </ul>
    <span slot="footer">Card footer</span>
  </bs-card>
</div>`} />

## Header and footer

Add a header via the `header` slot. A body follows by default.

<Example code={`<bs-card heading="Special title treatment">
  <span slot="header">Featured</span>
  <p class="card-text">With supporting text below as a natural lead-in to additional content.</p>
  <a href="#" class="btn btn-primary">Go somewhere</a>
</bs-card>`} />

Headers can include heading elements like `<h5 class="card-header">`.

<Example code={`<bs-card heading="Special title treatment">
  <h5 slot="header" class="card-header">Featured</h5>
  <p class="card-text">With supporting text below as a natural lead-in to additional content.</p>
  <a href="#" class="btn btn-primary">Go somewhere</a>
</bs-card>`} />

Headers pair nicely with blockquotes.

<Example code={`<bs-card>
  <span slot="header">Quote</span>
  <figure class="mb-0">
    <blockquote class="blockquote">
      <p>A well-known quote, contained in a blockquote element.</p>
    </blockquote>
    <figcaption class="blockquote-footer mb-0">
      Someone famous in <cite title="Source Title">Source Title</cite>
    </figcaption>
  </figure>
</bs-card>`} />

Cards support a footer via the `footer` slot too.

<Example code={`<bs-card heading="Special title treatment" class="text-center">
  <span slot="header">Featured</span>
  <p class="card-text">With supporting text below as a natural lead-in to additional content.</p>
  <a href="#" class="btn btn-primary">Go somewhere</a>
  <span slot="footer" class="text-body-secondary">2 days ago</span>
</bs-card>`} />

## Image overlays

Turn an image into a card background by rendering content into the
`img-overlay` slot. The image slot provides a `.card-img` (use
`class="card-img"` instead of `card-img-top`).

<Example code={`<bs-card class="text-bg-dark" no-body>

## Horizontal

Use the `horizontal` attribute to render the card with an image column on
the left and the body on the right. The image slot becomes the left column
(`.col-md-4`) and all other content fills `.col-md-8`.

<Example code={`<div class="mb-3" style="max-width: 540px;">
  <bs-card horizontal heading="Card title">

## API

| Attribute | Type | Default | Description |
| --- | --- | --- | --- |
| `variant` | `'primary' \| 'secondary' \| 'success' \| 'danger' \| 'warning' \| 'info' \| 'light' \| 'dark'` | — | Contextual `.text-bg-{variant}` background. |
| `text-variant` | Same as `variant` | — | Applies `.text-{variant}` to the card for bordered-card styles. |
| `heading` | `string` | — | Renders `<h5 class="card-title">` at the top of the body. |
| `subtitle` | `string` | — | Renders `<h6 class="card-subtitle">` below the title. |
| `no-body` | `boolean` | `false` | Skip the default `.card-body` wrapper — slot content becomes a direct child of the card. |
| `horizontal` | `boolean` | `false` | Render as a horizontal row (image-left, body-right). |

### Slots

| Slot | Description |
| --- | --- |
| *(default)* | Card body content. |
| `header` | Rendered in `.card-header`. |
| `footer` | Rendered in `.card-footer`. |
| `image` | Image rendered above the body (`.card-img-top`). |
| `image-bottom` | Image rendered after the footer (`.card-img-bottom`). |
| `img-overlay` | Content rendered inside `.card-img-overlay` on top of the image. |

See `packages/components/src/card/` in the source for the complete property list.
