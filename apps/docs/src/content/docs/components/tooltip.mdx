Floating text tooltip built on `@floating-ui/dom`. Wrap any trigger in
`<bs-tooltip>` and set the `content` attribute — the tooltip shows on
hover / focus by default and matches Bootstrap 5.3's visual styling.

## Overview

Things to know when using the tooltip component:

- Tooltips rely on `@floating-ui/dom` for positioning and are rendered
  inside the component's shadow DOM, so no global initialisation is
  needed — the element just works once imported.
- Trigger content goes in the default slot. Tooltip text goes in the
  `content` attribute.
- Tooltips don't fire on hidden elements. If the target host is
  `display: none` the tooltip never mounts.
- Keyboard and assistive-tech users get the tooltip via `focus` just
  like hover.

<Example code={`<bs-tooltip content="A helpful tooltip"><bs-button>Hover me</bs-button></bs-tooltip>`} />

## Examples

### Tooltips on links

Hover each of the inline links below to see its tooltip. This mirrors
Bootstrap's `data-bs-toggle="tooltip"` pattern — wrap each trigger in
a separate `<bs-tooltip>`.

<Example code={`<p class="muted">Placeholder text to demonstrate some <bs-tooltip content="Default tooltip"><a href="#">inline links</a></bs-tooltip> with tooltips. This is now just filler, no killer. Content placed here just to mimic the presence of <bs-tooltip content="Another tooltip"><a href="#">real text</a></bs-tooltip>. So hopefully you've now seen how <bs-tooltip content="Another one here too"><a href="#">these tooltips on links</a></bs-tooltip> can work in practice, once you use them on <bs-tooltip content="The last tip!"><a href="#">your own</a></bs-tooltip> site or project.</p>`} />

### Custom tooltips

You can customize the appearance of tooltips per-instance via the
`custom-class` attribute, which mirrors Bootstrap's
`data-bs-custom-class`. The class lands on the rendered tooltip
element, so any CSS variables defined for that class theme only that
tooltip.

<style>{`
  .custom-tooltip {
    --bs-tooltip-bg: #6f42c1;
    --bs-tooltip-color: var(--bs-white);
  }
`}</style>

<Example code={`<bs-tooltip content="This top tooltip is themed via CSS variables." placement="top" custom-class="custom-tooltip"><bs-button variant="secondary">Custom tooltip</bs-button></bs-tooltip>`} />

```css
.custom-tooltip {
  --bs-tooltip-bg: #6f42c1;
  --bs-tooltip-color: var(--bs-white);
}
```

### Directions

Set `placement` to `top` (default), `right`, `bottom`, or `left` to
position the tooltip around its trigger.

<Example code={`<div class="d-flex gap-2 flex-wrap">
  <bs-tooltip content="Tooltip on top" placement="top"><bs-button variant="secondary">Tooltip on top</bs-button></bs-tooltip>
  <bs-tooltip content="Tooltip on right" placement="right"><bs-button variant="secondary">Tooltip on right</bs-button></bs-tooltip>
  <bs-tooltip content="Tooltip on bottom" placement="bottom"><bs-button variant="secondary">Tooltip on bottom</bs-button></bs-tooltip>
  <bs-tooltip content="Tooltip on left" placement="left"><bs-button variant="secondary">Tooltip on left</bs-button></bs-tooltip>
</div>`} />

### HTML content

`<bs-tooltip>` renders the `content` attribute as plain text by
default, matching Bootstrap's default sanitiser behaviour. This keeps
tooltip content safe from accidental HTML injection.

<Example code={`<bs-tooltip content="Tooltip with HTML (rendered as text)"><bs-button variant="secondary">Tooltip with HTML</bs-button></bs-tooltip>`} />

### Directions

Any inline content can be the trigger — SVGs, icons, links, buttons.
The tooltip reference wrapper is `d-inline-block` so layout stays
predictable.

<Example code={`<bs-tooltip content="Default tooltip" placement="top"><a href="#" class="d-inline-block" aria-label="Hover or focus to see default tooltip"><svg xmlns="http://www.w3.org/2000/svg" width="50" height="50" viewBox="0 0 100 100" aria-hidden="true"><rect width="100%" height="100%" fill="#563d7c"></rect><circle cx="50" cy="50" r="30" fill="#007bff"></circle></svg></a></bs-tooltip>`} />

### Disabled elements

Elements with the `disabled` attribute don't fire mouse events, so
tooltips won't trigger directly. Wrap a disabled control in a focusable
`<span tabindex="0">` (or give that wrapper to `<bs-tooltip>`) to keep
the tooltip working.

<Example code={`<bs-tooltip content="Disabled tooltip"><span class="d-inline-block" tabindex="0"><bs-button variant="primary" disabled>Disabled button</bs-button></span></bs-tooltip>`} />

## Install

```sh
# as an npm package
npm install @bootstrap-wc/components
# or copy the source
npx bwc add tooltip
```

Additional peer deps: `@floating-ui/dom`

## Usage

```ts
import '@bootstrap-wc/components/tooltip';
```

```html
<bs-tooltip content="A helpful tooltip"><bs-button>Hover me</bs-button></bs-tooltip>
```

## API

| Attribute | Type | Default | Description |
| --- | --- | --- | --- |
| `content` | `string` | `""` | Tooltip text shown in the floating tip. |
| `placement` | `'top' \| 'right' \| 'bottom' \| 'left'` | `'top'` | Side of the trigger the tooltip renders on. |
| `trigger` | `'hover' \| 'focus' \| 'click' \| 'manual'` | `'hover'` | What opens/closes the tooltip. `hover` also opens on focus. |
| `show-delay` | `number` (ms) | `0` | Delay before showing. |
| `hide-delay` | `number` (ms) | `0` | Delay before hiding. |
| `open` | `boolean` | `false` | Controlled open state; reflects as attribute. |
| `custom-class` | `string` | `""` | Extra class applied to the rendered tooltip (for CSS-variable theming). |

See `packages/components/src/tooltip/` in the source for the complete property / event list.
