`<bs-navbar>` is the chrome of a Bootstrap navbar — it mirrors `.navbar`,
`.navbar-expand-{x}`, `bg-{x}`, `.navbar-dark` / `.navbar-light`, and the
placement class onto the host so author classes (`mb-4`, `shadow-sm`,
`rounded`) and Bootstrap's `.navbar > .container-fluid` selector all work
naturally on the host element.

The toggler, the `.collapse.navbar-collapse`, and any `.offcanvas` drawer
are **author-supplied light-DOM children**. They go directly inside
`<bs-navbar>` and are wired with the standard `data-bs-toggle` /
`data-bs-target` attributes, so every Bootstrap navbar pattern works
exactly as documented upstream — internal collapse, offcanvas, external
collapse target, responsive offcanvas, dropdowns. Load
`bootstrap.bundle.min.js` for the toggle behavior and the height-animation;
the pixel-diff comparator pages also pair it with `bs-nav` and `bs-button`
inside the collapse so the items themselves are framework components.

## Install

```sh
# as an npm package
npm install @bootstrap-wc/components
# or copy the source
npx bwc add navbar
```

## Usage

```ts
import '@bootstrap-wc/components/navbar';
```

```html
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js" defer></script>
```

## Supported content

A responsive navbar that collapses at the `lg` breakpoint with brand,
nav, and a search form.

<Example code={`<bs-navbar background="body-tertiary">
  <a class="navbar-brand" href="#">Navbar</a>
  <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#nav-supported" aria-controls="nav-supported" aria-expanded="false" aria-label="Toggle navigation">
    <span class="navbar-toggler-icon"></span>
  </button>
  <div class="collapse navbar-collapse" id="nav-supported">
    <bs-nav class="me-auto">
      <bs-nav-item href="#" active>Home</bs-nav-item>
      <bs-nav-item href="#">Link</bs-nav-item>
      <bs-nav-item disabled>Disabled</bs-nav-item>
    </bs-nav>
    <form class="d-flex" role="search">
      <bs-input type="search" placeholder="Search" aria-label="Search" class="me-2"></bs-input>
      <bs-button variant="success" button-style="outline" type="submit">Search</bs-button>
    </form>
  </div>
</bs-navbar>`} />

## Brand

Place an `<a class="navbar-brand">` (or a `<span>`) directly inside
`<bs-navbar>`. Bootstrap's `.navbar-brand` styling applies in light DOM.

### Text

<Example code={`<bs-navbar background="body-tertiary" expand="never">
  <a class="navbar-brand" href="#">Navbar</a>
</bs-navbar>`} />

### Image

<Example code={`<bs-navbar background="body-tertiary" expand="never">
  <a class="navbar-brand" href="#">
    <img src="https://getbootstrap.com/docs/5.3/assets/brand/bootstrap-logo.svg" alt="Bootstrap" width="30" height="24" />
  </a>
</bs-navbar>`} />

### Image and text

<Example code={`<bs-navbar background="body-tertiary" expand="never">
  <a class="navbar-brand d-inline-flex align-items-center" href="#">
    <img src="https://getbootstrap.com/docs/5.3/assets/brand/bootstrap-logo.svg" alt="Bootstrap" width="30" height="24" class="d-inline-block align-text-top me-2" />
    Bootstrap
  </a>
</bs-navbar>`} />

## Nav

Use `<bs-nav>` with `<bs-nav-item>` children for the primary navigation.
Hosts carry `.nav-link`, `.active`, and `.disabled` so Bootstrap's
selectors apply across the shadow boundary. When nested inside
`<bs-navbar>` `<bs-nav>` automatically picks up `.navbar-nav` too.

<Example code={`<bs-navbar background="body-tertiary" expand="always">
  <a class="navbar-brand" href="#">Navbar</a>
  <bs-nav>
    <bs-nav-item href="#" active>Home</bs-nav-item>
    <bs-nav-item href="#">Features</bs-nav-item>
    <bs-nav-item href="#">Pricing</bs-nav-item>
    <bs-nav-item disabled>Disabled</bs-nav-item>
  </bs-nav>
</bs-navbar>`} />

## Dropdown

Drop `<bs-dropdown>` straight into a navbar — items can be plain
`<bs-dropdown-item>` children. Since 0.6 they auto-default to
`slot="menu"`, so the snippet below works without needing to spell out
`slot="menu"` on every item.

<Example code={`<bs-navbar background="body-tertiary" expand="always">
  <a class="navbar-brand" href="#">Navbar</a>
  <bs-nav>
    <bs-nav-item href="#" active>Home</bs-nav-item>
    <bs-nav-item href="#">Link</bs-nav-item>
  </bs-nav>
  <bs-dropdown label="Dropdown" class="ms-auto">
    <bs-dropdown-item href="#">Action</bs-dropdown-item>
    <bs-dropdown-item href="#">Another action</bs-dropdown-item>
    <bs-dropdown-item divider></bs-dropdown-item>
    <bs-dropdown-item href="#">Something else</bs-dropdown-item>
  </bs-dropdown>
</bs-navbar>`} />

Nested dropdowns ("submenu" pattern) compose the same way. Apply
`class="dropdown-submenu"` to the inner `<bs-dropdown>` for the
right-flyout layout (consumer CSS, since Bootstrap 5 dropped native
submenu support).

<Example code={`<bs-navbar background="body-tertiary" expand="always">
  <bs-dropdown label="Apps" class="ms-auto">
    <bs-dropdown-item href="#">Inbox</bs-dropdown-item>
    <bs-dropdown class="dropdown-submenu" label="Project">
      <bs-dropdown-item href="#">Contacts</bs-dropdown-item>
      <bs-dropdown-item href="#">Calendar</bs-dropdown-item>
      <bs-dropdown-item href="#">Files</bs-dropdown-item>
    </bs-dropdown>
    <bs-dropdown-item href="#">Reports</bs-dropdown-item>
  </bs-dropdown>
</bs-navbar>`} />

## Forms

Drop a `<form>` into `<bs-navbar>` (typically inside the
`.collapse.navbar-collapse`) and Bootstrap's flex layout takes care of the
rest. Use `me-auto` on the leading nav to push the form to the right.

<Example code={`<bs-navbar background="body-tertiary" expand="always">
  <a class="navbar-brand" href="#">Navbar</a>
  <form class="d-flex ms-auto" role="search">
    <bs-input type="search" placeholder="Search" aria-label="Search" class="me-2"></bs-input>
    <bs-button variant="success" button-style="outline" type="submit">Search</bs-button>
  </form>
</bs-navbar>`} />

<Example code={`<bs-navbar background="body-tertiary" expand="always">
  <span class="navbar-text">Navbar text with an inline element</span>

<Example code={`<bs-navbar background="body-tertiary" expand="always">

  </bs-nav>
  <span class="navbar-text">Navbar text with an inline element</span>
</bs-navbar>`} />

## Color schemes

`theme="dark"` adds `.navbar-dark` to the host (`theme="light"` adds
`.navbar-light`). Both pull in Bootstrap's navbar-scoped dark / light CSS
variables for colors, borders, and the toggler icon — but unlike
`data-bs-theme="dark"` they don't cascade dark theme to slotted content
(so a `<bs-input>` inside a dark navbar keeps a light background, matching
upstream behavior). Combine with `background="*"` for the final hue.

<Example code={`<bs-navbar background="dark" theme="dark" expand="always">
  <a class="navbar-brand" href="#">Navbar</a>
  <bs-nav>
    <bs-nav-item href="#" active>Home</bs-nav-item>
    <bs-nav-item href="#">Features</bs-nav-item>
    <bs-nav-item href="#">Pricing</bs-nav-item>
  </bs-nav>
</bs-navbar>`} />

<Example code={`<bs-navbar background="primary" theme="dark" expand="always">
  <a class="navbar-brand" href="#">Navbar</a>
  <bs-nav>
    <bs-nav-item href="#" active>Home</bs-nav-item>
    <bs-nav-item href="#">Features</bs-nav-item>
    <bs-nav-item href="#">Pricing</bs-nav-item>
  </bs-nav>
</bs-navbar>`} />

<Example code={`<bs-navbar theme="light" style="background-color: #e3f2fd" expand="always">
  <a class="navbar-brand" href="#">Navbar</a>
  <bs-nav>
    <bs-nav-item href="#" active>Home</bs-nav-item>
    <bs-nav-item href="#">Features</bs-nav-item>
    <bs-nav-item href="#">Pricing</bs-nav-item>
  </bs-nav>
</bs-navbar>`} />

## Containers

The default container is `.container-fluid` (full width). Set
`container="default"` for the responsive `.container` (max-width per
breakpoint), `container="sm|md|lg|xl|xxl"` for a fixed-breakpoint
container, or `container="none"` to skip the inner wrapper entirely
when nesting `<bs-navbar>` inside your own outer `.container`.

<Example code={`<div class="container">
  <bs-navbar background="body-tertiary" container="none" expand="never">
    <a class="navbar-brand" href="#">Navbar (container outside)</a>
  </bs-navbar>
</div>`} />

<Example code={`<bs-navbar background="body-tertiary" container="default" expand="never">
  <a class="navbar-brand" href="#">Navbar (responsive container)</a>
</bs-navbar>`} />

<Example code={`<bs-navbar background="body-tertiary" container="md" expand="never">
  <a class="navbar-brand" href="#">Navbar (container-md)</a>
</bs-navbar>`} />

## Placement

The `placement` attribute pins the navbar to the viewport. Valid values
are `static` (default), `fixed-top`, `fixed-bottom`, `sticky-top` and
`sticky-bottom`. The legacy `fixed` / `sticky` booleans still work as
shorthands for `fixed-top` / `sticky-top`.

<Example code={`<bs-navbar background="body-tertiary" expand="never">
  <a class="navbar-brand" href="#">Default</a>
</bs-navbar>`} />

<Example code={`<bs-navbar background="body-tertiary" placement="sticky-top" expand="never">
  <a class="navbar-brand" href="#">Sticky top</a>
</bs-navbar>`} />

(`fixed-top` / `fixed-bottom` / `sticky-bottom` work the same way — they
pin the navbar to the viewport edge.)

## Scrolling

Add the `.navbar-nav-scroll` class to the slotted `<bs-nav>` to cap its
height and scroll overflow.

<Example code={`<bs-navbar background="body-tertiary" expand="always">
  <a class="navbar-brand" href="#">Navbar scroll</a>
  <bs-nav class="navbar-nav-scroll" style="--bs-scroll-height: 100px;">
    <bs-nav-item href="#" active>Home</bs-nav-item>
    <bs-nav-item href="#">Link</bs-nav-item>
    <bs-nav-item href="#">Link</bs-nav-item>
    <bs-nav-item href="#">Link</bs-nav-item>
    <bs-nav-item href="#">Link</bs-nav-item>
  </bs-nav>
</bs-navbar>`} />

## Responsive behaviors

Choose the breakpoint at which the navbar collapses with `expand`.
Acceptable values are `sm`, `md`, `lg` (default), `xl`, `xxl`, `always`
(never collapses) and `never` (always collapsed).

Below the breakpoint Bootstrap hides the `.collapse.navbar-collapse` and
shows the `.navbar-toggler`. Above the breakpoint the collapse becomes a
flex row and the toggler is hidden. Bootstrap's bundle JS animates the
height transition with the `.collapsing` class — there's no extra wiring
on `<bs-navbar>`'s side.

### Toggler

A toggler is a regular `<button class="navbar-toggler">` with
`data-bs-toggle="collapse"` and `data-bs-target="#…"` pointing at the
`.collapse` element it controls.

<Example code={`<bs-navbar background="body-tertiary" expand="lg">
  <a class="navbar-brand" href="#">Expand at lg</a>
  <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#nav-toggler-demo" aria-controls="nav-toggler-demo" aria-expanded="false" aria-label="Toggle navigation">
    <span class="navbar-toggler-icon"></span>
  </button>
  <div class="collapse navbar-collapse" id="nav-toggler-demo">
    <bs-nav class="me-auto">
      <bs-nav-item href="#" active>Home</bs-nav-item>
      <bs-nav-item href="#">Link</bs-nav-item>
      <bs-nav-item disabled>Disabled</bs-nav-item>
    </bs-nav>
  </div>
</bs-navbar>`} />

### External content

Point a toggler at any `.collapse` element on the page — it doesn't have
to live inside the navbar. The Album example template uses this pattern
to slide an About / Contact panel out from above the brand bar.

```html
<header>
  <div class="collapse text-bg-dark" id="navbarHeader">
    …about, contact, etc.
  </div>
  <bs-navbar background="dark" theme="dark" container="default" expand="never" class="shadow-sm">
    <a class="navbar-brand" href="#">Album</a>
    <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#navbarHeader" aria-controls="navbarHeader" aria-expanded="false" aria-label="Toggle navigation">
      <span class="navbar-toggler-icon"></span>
    </button>
  </bs-navbar>
</header>
```

### Offcanvas

Swap `data-bs-toggle="collapse"` for `data-bs-toggle="offcanvas"` and
target an `.offcanvas` element to slide the menu in from the side
instead of expanding inline. Pair with `expand="lg"` (or another
breakpoint) for the responsive-offcanvas pattern: drawer below the
breakpoint, inline column at and above it.

```html
<bs-navbar background="dark" theme="dark" expand="lg" container="fluid">
  <a class="navbar-brand" href="#">Responsive offcanvas navbar</a>
  <button class="navbar-toggler" type="button" data-bs-toggle="offcanvas" data-bs-target="#navOffcanvas" aria-controls="navOffcanvas" aria-label="Toggle navigation">
    <span class="navbar-toggler-icon"></span>
  </button>
  <div class="offcanvas offcanvas-end text-bg-dark" tabindex="-1" id="navOffcanvas" aria-labelledby="navOffcanvasLabel">
    <div class="offcanvas-header">
      <h5 class="offcanvas-title" id="navOffcanvasLabel">Offcanvas</h5>
      <button type="button" class="btn-close btn-close-white" data-bs-dismiss="offcanvas" aria-label="Close"></button>
    </div>
    <div class="offcanvas-body">
      <bs-nav class="justify-content-end flex-grow-1 pe-3">
        <bs-nav-item href="#" active>Home</bs-nav-item>
        <bs-nav-item href="#">Link</bs-nav-item>
      </bs-nav>
    </div>
  </div>
</bs-navbar>
```

## API

| Attribute     | Type                                                                          | Default   | Notes                                                                                            |
| ------------- | ----------------------------------------------------------------------------- | --------- | ------------------------------------------------------------------------------------------------ |
| `expand`      | `sm` \| `md` \| `lg` \| `xl` \| `xxl` \| `always` \| `never`                  | `lg`      | Breakpoint at which the navbar becomes horizontal.                                               |
| `theme`       | `light` \| `dark` \| `auto`                                                   | `auto`    | Adds `.navbar-light` / `.navbar-dark` to the host. `auto` inherits the document's color mode.    |
| `background`  | `primary` \| `dark` \| `body-tertiary` \| any Bootstrap bg utility            | `—`       | Adds `bg-{value}` to the host.                                                                   |
| `placement`   | `static` \| `fixed-top` \| `fixed-bottom` \| `sticky-top` \| `sticky-bottom`  | `static`  | Pins the navbar to the viewport.                                                                 |
| `container`   | `fluid` \| `default` \| `sm` \| `md` \| `lg` \| `xl` \| `xxl` \| `none`       | `fluid`   | Inner wrapper. `default` = `.container` (responsive max-width). `none` skips the wrapper.        |
| `fluid`       | `boolean`                                                                     | `true`    | Legacy shorthand — when `false` and `container` is unset, renders `.container` instead of fluid. |
| `fixed`       | `boolean`                                                                     | `false`   | Legacy shorthand for `placement="fixed-top"`.                                                    |
| `sticky`      | `boolean`                                                                     | `false`   | Legacy shorthand for `placement="sticky-top"`.                                                   |

| Slot      | Purpose                                                                                                |
| --------- | ------------------------------------------------------------------------------------------------------ |
| (default) | Everything — brand link, toggler, `.collapse.navbar-collapse`, `.offcanvas`. Projected through         |
|           | `[part="container"]` so Bootstrap's `.navbar > .container-fluid` flex layout applies.                  |

| Part        | Purpose                                                  |
| ----------- | -------------------------------------------------------- |
| `container` | The `.container-fluid` (or sized) wrapper. Suppress with `container="none"` to project directly. |

See `packages/components/src/navbar/` in the source for the implementation.
