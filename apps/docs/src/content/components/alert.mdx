## Live example

Toggle the `open` property (or use `close()`) to show and hide an alert without
re-creating the element. The button below flips `open` back to `true` and the
alert fades back in.

<Example code={`<div id="liveAlertPlaceholder"></div>
<bs-button variant="primary" onclick="(() => { const host = document.getElementById('liveAlertPlaceholder'); if (host.firstElementChild) return; const a = document.createElement('bs-alert'); a.setAttribute('variant', 'success'); a.setAttribute('dismissible', ''); a.textContent = 'Nice, you triggered this alert!'; a.addEventListener('bs-dismiss', () => a.remove()); host.appendChild(a); })()">Show live alert</bs-button>`} />

## Additional content

Alerts accept rich content. Use the `heading` slot for an `.alert-heading`
element, and mix in paragraphs, horizontal rules and utility classes as needed.

<Example code={`<bs-alert variant="success">
  <h4 slot="heading" class="alert-heading">Well done!</h4>
  <p>Aww yeah, you successfully read this important alert message. This example text is going to run a bit longer so that you can see how spacing within an alert works with this kind of content.</p>
  <hr />
  <p class="mb-0">Whenever you need to, be sure to use margin utilities to keep things nice and tidy.</p>
</bs-alert>`} />

## Dismissing

Add the `dismissible` attribute to render a close button. The alert fades out
and dispatches `bs-dismiss` when closed.

<Example code={`<bs-alert variant="warning" dismissible>
  <strong>Holy guacamole!</strong> You should check in on some of those fields below.
</bs-alert>`} />

## API

### Attributes / properties

| Name | Type | Default | Description |
|---|---|---|---|
| `variant` | Bootstrap variant | `primary` | Color. |
| `dismissible` | `boolean` | `false` | Show close button. |
| `open` | `boolean` | `true` | Controls visibility (reflected). |
| `dismiss-label` | `string` | `'Close'` | ARIA label for dismiss button. |

### Slots

| Name | Description |
|---|---|
| _(default)_ | Alert content. Accepts rich markup — paragraphs, links (`.alert-link`), flex containers with icons, etc. |
| `heading` | Optional heading block, typically an `<h4 class="alert-heading">`. |

### Methods

| Method | Description |
|---|---|
| `close()` | Programmatically close with fade animation. |

### Events

| Event | Description |
|---|---|
| `bs-dismiss` | Fired after the alert finishes closing. |
